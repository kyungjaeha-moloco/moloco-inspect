## 주의사항 (지뢰)

1. **현재 canvas-app dev 서버 (:4180)** 가 떠있을 수 있음. M2 리팩터 중엔 kill하거나 playground-app을 **다른 포트 (:4181 권장)** 로 돌릴지 결정. 제안: 디렉토리 rename 후 `:4180` 그대로 사용, canvas-app은 죽음.
2. **localStorage 키 변경 (v2 → v3)** 가 일어날 때 현재 사용자 (본인)의 기존 Canvas 데이터가 있다면 migrate 필요. 없다면 skip 로직만 동작.
3. **iframe src 포트** 는 실시간으로 playground API에서 받아야 함. 고정 값 하드코딩 금지.
4. **Vite HMR WebSocket** 이 iframe 안에서 정상 동작하는지 초기에 확인. 안 되면 `iframe key` bump로 재로드 폴백.
5. **orchestrator 재기동** 시 포트는 바뀔 수 있음. UI가 매번 `/api/playground/:id` 재호출해야.
