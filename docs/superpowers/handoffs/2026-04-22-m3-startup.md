## M3 범위 (v3 plan §5 + §14 E4)

### 목표

Vite 플러그인 picker 구현 + postMessage nonce handshake로 iframe과 parent를 묶어서:

1. Pick 모드에서 실제 React element 선택 가능
2. Pin 코멘트가 좌표 대신 **element 식별자**에 묶임
3. iframe SPA 라우트를 parent가 실시간 감지 → pin 필터링 정확도 복원

### 산출물

- `sandbox/vite-plugin-playground-picker/` (신규 패키지, M3 Task #1)
  - 브라우저 번들에 주입되는 picker 런타임
  - Hover 시 element 하이라이트 (outline overlay)
  - 클릭 시 fiber walker로 식별자 수집:
    1. `data-testid` (최우선)
    2. React fiber `displayName`
    3. `_debugSource` → sourceFile:line (dev-only, best-effort)
    4. CSS selector path (fallback)
  - postMessage로 parent에 통지
  - SPA 라우트 변화 감시 (history.pushState/popstate) → parent 알림

- `playground-app/src/services/playground-bridge.ts` (신규)
  - iframe으로부터 postMessage 수신 핸들러
  - nonce 발급/검증 (v3 §5.2)
  - allowlist origins 체크 (`http://127.0.0.1:*`, `http://localhost:*`)
  - 수신 이벤트 → `pin-store`, `playground-store` 업데이트

- `element-picker-core` 패키지 분리 (Chrome Ext와 공유 예정 — M4)
  - fiber walker + 식별자 우선순위 로직만 추출

### v3 plan의 M3 체크리스트

플랜 §9 표 기준 M3 완료 기준:

## M3 시작 순서 (추천)

0. **환경 확인** (위 "현재 상태")
1. **M0 스파이크 리포트 재확인** — `docs/spikes/2026-04-22-playground-feasibility.md` E4 섹션에서 fiber/sourceFile/testId 수집률 숫자 확인. 낮으면 fallback 전략 선언
2. **플러그인 스켈레톤** — `sandbox/vite-plugin-playground-picker/` 에 Vite plugin 기본 셋업 (transformIndexHtml 에 `<script>` 주입 또는 virtual module)
3. **Picker 런타임** — hover/click 핸들러 + fiber walker. 일단 console.log로 식별자 찍어보고 검증
4. **Sandbox Dockerfile 갱신** — `playground-app` 빌드 시점에 picker 포함되게. `v3-playground` 이미지 재빌드 + 태그 `v3-playground-m3`
5. **Parent bridge** — `playground-app/src/services/playground-bridge.ts` 로 postMessage 수신. nonce 검증 (v3 §5.2)
6. **iframe 주입 확인** — Playground 열었을 때 iframe 내부에서 picker 런타임 로딩 + handshake 성공
7. **Pick 모드 연결** — iframe에서 element 클릭 → parent가 `ElementContext` 받음 → playground-store에 `lastPickedElement` 로 노출
8. **Pin 모드 통합** — Pin 모드에서도 hover 가능하게 + 클릭 시 `element` 필드 채우기 (기존 `x, y` 좌표는 보조로 유지)
9. **SPA 라우트 추적** — picker가 `history.pushState` 패치 → 변경 시 parent 통지 → `playground-store.currentRoute` 업데이트 → `LivePreview` pin 필터링 조건 갱신
10. **Chrome Ext 공유 준비 (선택)** — picker 런타임 / element-picker-core 분리. M4에서 바로 import 가능하게

각 단계 끝날 때: orchestrator 재기동 불필요, 샌드박스 이미지만 재빌드하면 됨 (`docker build -t moloco-inspect-sandbox:v3-playground-m3 sandbox/`). Playground는 재생성하거나 기존 컨테이너 archive 후 새 이미지로 create.

## 주의사항 (지뢰)

1. **샌드박스 이미지 재빌드 필수** — picker 런타임이 이미지 안에 들어가야 함. 현 실행 중인 9cb08297 container는 이전 이미지라 picker 없음. 테스트는 새 playground 생성으로.
2. **nonce 유출 시 피해 범위** — v3 §10 R17. Playground 하나의 세션 단위. iframe 세션 reload 시 새 nonce 발급.
3. **`pointerEvents` 상호작용** — Pick 모드에서는 parent 오버레이 제거해야 iframe으로 이벤트 전달됨. 기존 코드 (`LivePreview.tsx` `showOverlay = mode !== 'pick'`) 이미 반영됨 — 확인만.
4. **fiber 접근 private API** — `__reactFiber$xxx` / `__reactProps$xxx` 키는 minified prop names. React 버전 업그레이드 시 깨질 수 있음. v3 §14 E4에 우선순위 전략 — testid 최우선, fiber는 best-effort.
5. **Cross-origin iframe 제약** — parent가 iframe.contentWindow.location 읽기는 여전히 막힘. 모든 route 정보는 postMessage로만.
6. **M1b #6 lifecycle test** — 아직 미작성. M3 리팩터하면서 깨질 가능성 큼. 여유 있을 때 v3 §8.5 스펙대로 `orchestrator/test/lifecycle.test.js` 작성 권장.
