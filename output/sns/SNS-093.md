### [LINKEDIN]
AI가 만든 UI 변경사항을 공유할 때 "링크 보내드렸습니다"로 끝나면 안 됩니다.

Preview proxy를 만들었습니다. AI가 생성한 코드 변경 결과를 실제 데이터와 세션 맥락이 유지된 상태로 미리 볼 수 있는 환경입니다.

왜 맥락 보존이 중요한가. 빈 화면에서 컴포넌트만 보면 실제 사용 환경에서 어떻게 보이는지 알 수 없습니다. "이 버튼이 실제 데이터 100개 있을 때도 잘 보이나?" 같은 질문에 답하려면 실제 맥락이 필요합니다.

Approve 결정은 미리보기의 품질에 달려 있습니다. 맥락이 빠진 미리보기는 잘못된 승인을 만들 수 있습니다.

결과물 공유와 미리보기를 어떻게 처리하고 계신가요?
