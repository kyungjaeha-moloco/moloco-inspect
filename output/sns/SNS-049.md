### [LINKEDIN]
다크 모드를 구현할 때 얼마나 많은 곳을 수정해야 했나요?

저는 디자인 시스템 사이트에 다크 모드를 추가하면서 CSS 변수 60곳을 수정했습니다. 60곳이라는 숫자는 나쁜 숫자가 아닙니다. 하드코딩된 곳이 없다는 의미이기 때문입니다.

토큰 시스템이 잘 갖춰져 있으면 다크 모드는 단순한 변수 값 교체 작업입니다. `--color-background: #ffffff` → `--color-background: #0f172a`. CSS 변수 60개를 `[data-theme="dark"]` 선택자 아래 재정의하면 전체 사이트가 다크 모드로 전환됩니다.

반대로 하드코딩이 섞여 있으면 다크 모드 작업은 끝이 없습니다. 60곳이 아니라 600곳, 그것도 찾기도 어렵게 흩어져 있습니다.

다크 모드 지원 비용은 지금 토큰을 얼마나 잘 써왔느냐에 달려 있습니다.
