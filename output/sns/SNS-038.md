### [TWITTER]
AI가 에러 메시지를 "An error has occurred."라고 썼다.

우리 서비스 톤이 아니다.

ux-writing.json에 톤, 금지 표현, 용어 대체어를 정리했더니 AI가 우리 말투로 쓰기 시작했다.

#UXWriting #DesignSystem

### [LINKEDIN]
AI가 생성하는 UI 텍스트에서 가장 자주 문제가 되는 것은 무엇일까요?

저의 경험상 "톤"입니다. 기술적으로 정확하지만, 우리 서비스의 말투가 아닌 텍스트들이 나옵니다. "An error has occurred. Please try again." 이런 차가운 시스템 메시지처럼요.

`ux-writing.json`을 만들었습니다. 서비스 전체의 톤 원칙, 상황별 금지 표현과 권장 표현, 핵심 용어의 한국어/영어 표기 기준을 JSON으로 구조화했습니다.

예를 들어: `"error_messages": { "tone": "공감적, 해결 지향적", "avoid": ["에러가 발생했습니다", "오류"], "prefer": ["문제가 생겼어요", "다시 시도해주세요"] }`

AI가 이 파일을 참조하면 텍스트 생성 시 톤과 용어 기준을 따릅니다. 카피라이터 없이도 일관된 톤이 유지됩니다.

UX 라이팅도 시스템으로 만들 수 있습니다.

서비스 톤 가이드, 어떻게 관리하고 계신가요?
