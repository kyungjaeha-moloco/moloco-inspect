### [LINKEDIN]
AI가 만든 결과물이 한 번에 완벽할 거라는 기대를 버렸습니다.

대신 반복 루프를 설계했습니다. Request Changes → AI 재시도 → 리뷰 → 최대 3회.

사용자가 "이 부분이 맞지 않습니다"라고 피드백을 남기면 AI가 해당 피드백을 반영해서 다시 시도합니다. 이 과정을 최대 3회까지 허용하고, 3회 후에도 해결되지 않으면 사람에게 에스컬레이션합니다.

왜 3회인가. 1회는 너무 적고, 무한은 사용자가 언제 끝나는지 모릅니다. 3회라는 제한이 "이 정도면 충분히 시도했다"는 심리적 마감을 줍니다.

AI 협업에서 반복을 어떻게 구조화하고 계신가요?
