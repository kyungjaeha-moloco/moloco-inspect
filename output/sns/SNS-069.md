### [LINKEDIN]
Mock 토큰으로 자동화 테스트를 작성했습니다.

유닛 테스트에서는 잘 됐어요. 실제 API를 호출하는 통합 테스트에서는 401이 계속 났습니다. Mock 토큰은 API 서버가 검증할 수 있는 실제 JWT가 아니었으니까요.

두 가지 선택지가 있었어요.

첫째, API를 Mock으로 대체한다. 빠르지만 실제 동작을 검증하지 못해요.

둘째, 테스트 환경에서 실제 토큰을 자동 발급한다. 복잡하지만 실제 흐름을 검증할 수 있어요.

두 번째를 선택했습니다. 테스트 시작 시 인증 엔드포인트에 테스트 계정으로 로그인하고, 실제 JWT를 받아 이후 요청에 사용하는 방식으로요.

테스트 환경이 프로덕션 환경처럼 동작하기 시작했습니다. 실제로 실패하면 실제 이유로 실패했어요. 이게 맞는 테스트라고 생각했어요.
