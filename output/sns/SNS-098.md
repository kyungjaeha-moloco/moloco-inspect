### [LINKEDIN]
"좋은 인프라를 쓰면 좋은 결과가 나온다"는 가정이 틀렸습니다.

AI 에이전트를 운영하면서 가장 크게 결과를 바꾼 것은 서버 성능도, 모델 버전도 아니었습니다. 프롬프트였습니다.

같은 Claude 모델, 같은 코드베이스, 같은 태스크. 프롬프트에 "이 프로젝트는 React + TypeScript입니다. 컴포넌트는 함수형으로 작성하고, 스타일은 Tailwind를 씁니다. 에러는 반드시 try-catch로 처리하세요"를 추가했을 때와 없을 때의 코드 품질 차이는 명확했습니다.

컨텍스트를 잘 설계하는 것이 AI 시대의 설계 역량입니다. 어떤 정보를 주고, 어떤 제약을 걸고, 어떤 예시를 보여주느냐. 이것이 결과의 품질을 결정합니다.

프롬프트 설계에서 가장 효과적이었던 방법이 무엇인지 궁금합니다.
