### [TWITTER]
AI가 디자인 시스템을 직접 쿼리한다.

MCP 서버에 9개 도구를 붙였다. 컴포넌트 검색, 토큰 조회, 사용 예시 조회까지.

이제 AI가 Figma 링크 달라고 안 한다.

#MCP #DesignSystem

### [LINKEDIN]
"이 컴포넌트 스펙이 어떻게 되나요?" AI가 물어볼 때마다 링크를 복사해서 붙여넣었습니다.

MCP(Model Context Protocol) 서버를 디자인 시스템에 연결하고 나서 이 반복 작업이 사라졌습니다.

9개의 도구를 만들었습니다.

- `search_component`: 컴포넌트 이름으로 검색
- `get_token`: 토큰 이름으로 값 조회
- `list_variants`: 컴포넌트의 모든 변형 목록
- `get_usage_examples`: 실제 사용 예시 조회
- `check_deprecated`: deprecated 여부 확인
- 그 외 4개

AI가 컴포넌트 정보가 필요할 때 직접 디자인 시스템 MCP 서버에 쿼리합니다. 제가 중간에서 정보를 전달할 필요가 없습니다.

AI와 디자인 시스템 사이의 인터페이스를 설계했더니 제가 그 인터페이스에서 빠져도 되게 됐습니다.

여러분의 디자인 시스템은 AI가 직접 쿼리할 수 있나요?
