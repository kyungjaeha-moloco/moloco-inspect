### [LINKEDIN]
컴포넌트 상세 페이지가 764줄이었습니다.

Overview 탭, Props 탭, Usage 탭, Code 탭… 8개 탭의 로직이 전부 한 파일에 들어 있었어요. 어느 탭을 수정하려면 그 탭이 어디서 시작하는지 찾는 것부터 시작해야 했습니다.

8개 탭을 각각 독립 파일로 분리했습니다. ComponentDetailPage는 이제 탭을 조립하는 역할만 합니다. 245줄.

흥미로웠던 건 AI의 반응이었습니다. 분리 전에는 "이 탭 수정해줘"라고 하면 파일 전체를 읽고 관련 없는 코드까지 건드렸어요. 분리 후에는 정확히 그 탭 파일만 열었습니다.

파일 구조가 AI의 주의를 안내합니다. 이게 리팩토링의 숨겨진 효과였어요.
