### [LINKEDIN]
AI 도구 대시보드를 벤치마킹했습니다. Helicone, GitHub Copilot, Vercel 세 곳입니다.

각각 다른 데이터를 다룹니다. Helicone은 LLM API 비용과 응답 품질, Copilot은 코드 제안 수락률과 생산성, Vercel은 배포 상태와 성능 지표입니다.

공통점을 발견했습니다. 숫자를 나열하지 않는다는 것입니다. 숫자에 맥락을 붙입니다. "이번 주 토큰 사용량이 지난 주보다 23% 증가했습니다"가 아니라, "이 모델은 응답 시간이 빠르지만 비용이 높습니다. 대안을 검토하시겠습니까?"처럼요.

데이터 대시보드를 설계할 때 "무엇을 보여줄 것인가"보다 "사용자가 이 데이터로 무엇을 결정할 것인가"를 먼저 물어야 합니다.

AI 도구 대시보드에서 가장 유용하다고 느낀 지표가 무엇인지 궁금합니다.
