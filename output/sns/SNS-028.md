### [LINKEDIN]
"API 응답 필드명이 바뀌었어요. UI 다 깨졌어요."

이 대화, 한 번쯤 겪어보셨을 겁니다.

API와 UI가 암묵적으로 연결되어 있을 때 발생하는 문제입니다. 어디서 무엇이 연결되어 있는지 아무도 명시하지 않은 채로 개발이 진행됩니다.

저는 `api-ui-contracts.json`을 만들었습니다. API의 각 필드가 어떤 UI 컴포넌트의 어떤 속성으로 매핑되는지 명시한 계약서입니다. 예를 들어 `campaign.status` 필드는 `StatusBadge` 컴포넌트의 `variant` prop으로 연결되고, 허용 값은 `active | paused | ended`임을 명시합니다.

이 파일을 AI에게 읽히자, API 스펙 변경 요청이 들어왔을 때 영향받는 UI 컴포넌트 목록을 먼저 뽑아줬습니다.

계약서를 만들면 불일치가 드러납니다.

API↔UI 사이의 암묵적 연결, 지금 어떻게 관리하고 계신가요?
