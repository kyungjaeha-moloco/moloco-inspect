### [LINKEDIN]
디자인 시스템 사이트를 공유했을 때 돌아온 첫 번째 질문이 있었습니다.

"소스코드 어디서 볼 수 있어요?"

GitHub 링크가 없었어요. 컴포넌트 문서, 프리뷰, 코드 스니펫은 있는데 원본 저장소로 가는 길이 없었습니다. 사이트를 믿으려면 코드를 볼 수 있어야 한다는 걸 그때 깨달았어요.

Overview 섹션에 GitHub 링크를 추가했습니다. 한 줄의 변경이었지만, 그 이후로 같은 질문이 들어오지 않았어요.

디자인 시스템은 도구가 아니라 약속입니다. 이 컴포넌트가 어떻게 동작하는지, 어떻게 만들어졌는지 투명하게 공개하는 것이 신뢰의 시작이라고 생각합니다.

오픈 소스 프로젝트에서 투명성을 어떻게 표현하고 계신가요?

#DesignSystem #오픈소스 #ProductDesign #개발자경험
