### [LINKEDIN]
버튼 텍스트가 사용자의 확신에 영향을 줍니다.

"Approve & Apply" 버튼을 "✓ Approve & Create PR"로 바꿨습니다.

"Apply"는 모호합니다. 누르면 무슨 일이 생기는지 알 수 없습니다. 코드가 바로 배포되는 건지, 파일이 덮어써지는 건지, 아니면 다른 무언가인지.

"Create PR"은 다릅니다. GitHub Pull Request가 생성된다는 뜻을 개발자라면 바로 이해합니다. 되돌릴 수 없는 일이 일어나지 않는다는 안도감도 줍니다. PR을 통해 리뷰가 가능하니까요.

버튼 레이블은 액션의 이름이 아니라, 다음에 일어날 일의 예고여야 합니다. 기대값을 설정하는 것이 두려움을 줄이는 방법입니다.

버튼 텍스트 하나로 사용자 행동이 바뀐 경험이 있으신가요?
