### [LINKEDIN]
비개발자가 코드를 수정할 수 있는 플로우를 설계했습니다.

Select → Describe → Preview → PR. 4단계입니다.

1. Select: 어떤 파일/컴포넌트를 바꿀지 고른다
2. Describe: 자연어로 원하는 변경을 설명한다
3. Preview: AI가 만든 결과를 샌드박스에서 미리 본다
4. PR: 승인하면 자동으로 PR이 생성된다

이 여정에서 사용자는 코드를 한 줄도 보지 않아도 됩니다. 디자이너가 여정을 설계하면, AI가 실행하고, 개발자가 리뷰합니다.

역할의 경계가 바뀌고 있습니다. 코드를 모른다는 이유로 아이디어가 막히지 않는 환경을 만드는 것이 지금 제가 하고 있는 일입니다.

이런 플로우가 실제로 팀에 도입된다면 어떤 변화가 생길 것 같으신가요?
