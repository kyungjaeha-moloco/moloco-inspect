### [LINKEDIN]
새 제품을 만들기 전에 저는 항상 벤치마킹을 합니다.

Inspect를 설계하기 전, Helicone, GitHub Copilot, Vercel을 깊게 들여다봤습니다.

Helicone에서는 AI 사용량 모니터링의 시각화 방식을 배웠습니다. 복잡한 데이터를 어떻게 직관적으로 표현하는지. GitHub Copilot에서는 개발자 워크플로우와의 자연스러운 통합 방식을 배웠습니다. 흐름을 끊지 않는 도구의 설계. Vercel에서는 배포 상태와 로그를 실시간으로 보여주는 대시보드 구조를 배웠습니다. 상태 정보의 계층화.

벤치마킹은 따라 하기가 아닙니다. 이미 검증된 패턴에서 원리를 추출하고, 우리 제품의 맥락에 맞게 재해석하는 작업입니다.

레퍼런스 없이 만들면 이미 해결된 문제를 처음부터 다시 풀게 됩니다.

여러분은 새 제품을 설계할 때 어떤 방식으로 벤치마킹을 하시나요?
