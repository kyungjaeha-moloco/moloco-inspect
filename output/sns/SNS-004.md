### [TWITTER]
내가 설계하고, AI가 구현한다.

이게 말은 쉬운데 실제로 어떻게 작동하냐고?

나는 "무엇을" 결정하고, AI는 "어떻게"를 실행한다.
분업이 생각보다 명확하다.

---

### [LINKEDIN]
실제로 어떻게 분업이 이루어지는지 물어보는 분들이 많습니다.

제가 하는 일: 어떤 컴포넌트가 필요한지 결정하기, 데이터 흐름 설계하기, 사용자 시나리오 정의하기, 우선순위 판단하기, 결과물 검토하고 방향 수정하기.

AI가 하는 일: 코드 작성, 파일 구조 생성, API 연결, 오류 수정, 반복 작업 처리.

핵심은 "판단"과 "실행"의 분리입니다. 판단은 컨텍스트와 의도를 이해해야 하는 일이고, 실행은 그 판단을 구체화하는 일입니다. 디자이너는 판단에 강하고, AI는 실행에 강합니다.

이 분업이 자연스럽게 이루어질 때, 생산성이 비선형적으로 올라가는 걸 경험했습니다.

여러분은 AI와 협업할 때 어떤 부분을 직접 하고, 어떤 부분을 맡기시나요?

#ProductDesign #AIWorkflow #DesignProcess #협업
