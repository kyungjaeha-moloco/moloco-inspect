### [LINKEDIN]
Docker 이미지를 빌드할 때마다 `pnpm install`이 30초씩 걸렸습니다.

소스 파일 하나 바꾸고 빌드하면, 의존성은 하나도 안 바뀌었는데 패키지를 전부 다시 받았어요. 매번.

원인은 Dockerfile의 COPY 순서였습니다.

```
COPY . .          # 소스 전체 복사
RUN pnpm install  # 의존성 설치
```

소스 파일이 조금이라도 바뀌면 첫 번째 레이어가 무효화되고, 이후 레이어가 전부 다시 실행됩니다.

순서를 바꿨습니다.

```
COPY package.json pnpm-lock.yaml ./
RUN pnpm install
COPY . .
```

이제 `package.json`이 바뀌지 않으면 `pnpm install` 레이어가 캐시에서 옵니다. 1초.

Docker 레이어 캐싱은 순서가 전부입니다. 자주 바뀌는 것은 나중에, 자주 안 바뀌는 것은 먼저.
