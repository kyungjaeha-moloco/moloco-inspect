### [LINKEDIN]
AI가 코드를 생성할 때 네이밍 일관성이 깨지는 경험 해보셨나요?

같은 팀에서 같은 AI를 쓰더라도, 컴포넌트 이름, 파일 이름, prop 이름이 제각각이 되는 경우가 생깁니다. ButtonPrimary, PrimaryButton, primary-button, PrimaryBtn — 이 중 어느 것이 맞는지 알 수 없는 상황이 됩니다.

`conventions.json`은 네이밍 규칙과 파일 구조 규칙을 JSON으로 명시한 파일입니다.

컴포넌트 파일: PascalCase, 단수형
CSS 클래스: kebab-case
Prop 이름: camelCase, boolean은 is/has 접두어
파일 위치: `components/[category]/[ComponentName]/index.tsx`

AI가 새 파일을 만들기 전에 이 파일을 참조하면 일관된 구조가 만들어집니다. 나중에 사람이 수정할 필요가 없습니다.

규칙을 말로 설명하는 것과 JSON으로 명시하는 것의 차이입니다.

네이밍 컨벤션, 팀에서 어떻게 지키고 계신가요?
