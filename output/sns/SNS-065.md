### [LINKEDIN]
토큰이 만료됐는데 만료됐다는 걸 몰랐습니다.

API 응답에는 `expiresAt` 필드가 있었어요. 코드에서는 `expireTime`을 읽고 있었습니다. 둘 다 조용했어요. API는 필드를 보냈고, 코드는 다른 이름으로 읽으려다 `undefined`를 받았습니다. `undefined`는 만료 체크 로직에서 "만료되지 않음"으로 해석됐고요.

버그 자체는 한 줄 수정이었습니다. `expireTime` → `expiresAt`.

무서운 건 이 버그가 에러를 내지 않았다는 점입니다. 조용히 틀렸어요. 테스트에서 잡히려면 실제 만료 타임스탬프로 테스트해야 했는데, 목 데이터에는 늘 유효한 값이 있었으니 통과했죠.

필드명 불일치는 TypeScript에서 잡을 수 있었습니다. API 응답 타입을 정의했다면 `expireTime`을 쓰는 순간 컴파일 에러가 났을 거예요.
